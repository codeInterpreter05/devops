# Day 116 — Resources: MLOps Fundamentals

## Primary (assigned)

- **MLflow documentation** (mlflow.org/docs/latest) — the assigned starting point; covers Tracking, Projects, Models, and Model Registry with runnable examples for every ML framework it supports.

## Deepen your understanding

- **"Machine Learning Design Patterns" by Lakshmanan, Robinson & Munn (O'Reilly)** — pattern-based coverage of reproducibility, feature engineering, and model deployment concerns that map directly onto today's topics.
- **Google's "MLOps: Continuous delivery and automation pipelines in machine learning"** (cloud.google.com/architecture/mlops-continuous-delivery-and-automation-pipelines-in-machine-learning) — the widely-cited MLOps maturity model (levels 0/1/2) for how automated an ML pipeline actually is.
- **Feast documentation** (docs.feast.dev) — feature store concepts, point-in-time joins, and online/offline store architecture in depth.
- **Evidently AI docs/blog** (evidentlyai.com/docs) — practical, open-source-tool-grounded coverage of data and concept drift detection.
- **DVC documentation — Get Started** (dvc.org/doc/start) — the fastest hands-on path to understanding Git-like dataset versioning.

## Reference / lookup

- **MLflow Model Registry API reference** (mlflow.org/docs/latest/model-registry.html) — alias vs. stage APIs, exact method signatures.
- **DVC command reference** (dvc.org/doc/command-reference) — every `dvc` subcommand.

## Practice

- **MLflow's own quickstart tutorials** (mlflow.org/docs/latest/getting-started) — runnable end-to-end examples that mirror today's lab almost exactly, good for cross-checking your own setup.
