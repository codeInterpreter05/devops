## Deepen your understanding

- **`gitleaks`** (github.com/gitleaks/gitleaks) — a free, fast secret-scanning tool; run it against every repo you plan to pin before publicizing it, rather than relying on a manual `grep` pass alone.
- **GitHub Docs — "Removing sensitive data from a repository"** (docs.github.com) — the official, correct procedure for scrubbing a leaked credential from git history (via `git filter-repo` or BFG), not just deleting the file in a new commit.
- **"How to write a good README" guides (e.g., Make a README, makeareadme.com)** — a concise reference for structuring the what/why/how/tradeoff format recommended in today's portfolio audit.
- **Google SRE Workbook — "Postmortem Culture"** (sre.google/workbook) — useful background on what makes a debugging write-up genuinely valuable versus superficial, directly applicable to the "evidence of debugging" portfolio artifacts recommended today.

## Reference / lookup
